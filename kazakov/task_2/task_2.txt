                Различия между абстрактным классом и интерфейсом:

1.~ "Абстрактный класс" нужен для того, чтобы определить основные методы и поля для дочернего класса, так как от данного типа метдов нельзя создавать объекты.
Причём методы могут быть как абстрактные(не задаются), так и обычные(задаются):

abstract class Person{
    // поля...

    // конструктор

abstract void method();     // абстрактный метод

public void method2(...){...}   // обычный метод

}

  ~ "Интерфейс" нужен для обозначения определенных методов. Методы задаются внутри классов, имплементирующих данные интерфейсы, с помощью аннотации:

interface Interface1{
    void pick();
}

public class Human extends Person implements Interface1, Interface2 ... InterfaceN{     // класс, который имплементирует различные интерфейсы и наследует класс Person
    // поля...

    // конструктор

    @Override   // аннотация
    public void pick(...){...}   // задаём метод
}

2.~Нельзя наследовать несколько "классов"(любых).
  ~"Интерфейсы" можно наследовать сколько угодно.
